\section{Dynamics}
\label{sec:semantics}

This section presents a small-step operational semantics for the core language of
\cref{fig:syntax}.  

%\Cref{fig:values-evaluation-contexts} contains the
% basic definitions.
We classify values $\U, \V$ as expressions that are the usual weak-head normal forms: constants, variables
acting as channel names,
abstractions, injections of values, and pairs of values. Evaluation
contexts $\EC$ are standard except for the cases for directed application where the direction changes the evaluation order.  A process-local context $\FC$ embeds an evaluation context in a
single running process and process contexts $\CC$ close under parallel
composition and restriction.
Formal specifications for all four categories are shown in \cref{fig:values-evaluation-contexts}.

We omit the presentation of call-by-value expression reduction
$\E[1]\EReduce\E[2]$, found in the supplement in
\cref{fig:expression-reduction}. It includes the standard
computational steps for beta-reduction, sequencing, pair and variant
elimination, recursive unfolding, and closure under expression contexts.

Process congruence $\Proc\Congruent\ProcQ$ is defined as the standard structural congruence on
configurations: parallel composition is associative and commutative with unit
$\PExp{\CUnit}$, independent channel restrictions commute, and restrictions may
move across parallel composition. 
Its rules, found in the supplement in \cref{fig:configuration-congruence}, are used only to rearrange a
configuration so that a redex is exposed.

\Cref{fig:process-reduction} gives the process reduction relation $\Proc\PReduce\ProcQ$.
There are three groups of rules: ordinary computation inside a process, synchronization
between the two endpoints of a channel, and structural closure.

Rule \RuleName{R-Exp} embeds an expression step in the selected process-local context.
Rule \RuleName{R-New} introduces a fresh channel restriction and returns the two
endpoints as a pair.  Each endpoint is placed behind an initial separator
in its binder group, matching the acquiring session type assigned to the
restriction introduced by channel creation in \cref{fig:process-typing}.  Rule \RuleName{R-Fork} spawns the
thunked process in parallel and returns unit to the forking process.

Communication rules operate on the heads of dual binder groups.  Rule
\RuleName{R-Com} synchronizes send and receive, transfers the transmitted value,
and advances both groups.  Rule \RuleName{R-Choice} similarly synchronizes
selection and branching, returning the selected continuation as a variant.  Rule
\RuleName{R-Close} synchronizes close and wait at the end of a session, and rule
\RuleName{R-Discard} consumes an endpoint whose type permits it to be skipped.

The split rules implement the local and remote structure described in
\cref{sec:direct-semant-chann}.  Rule \RuleName{R-LSplit} replaces one endpoint
name by two names in the same binder group and returns the ordered
pair.  Because a binder group is a sequence, subsequent reductions must consume
the left component before the right component.  Rule \RuleName{R-RSplit} replaces
one endpoint name by two names separated by a binder-group separator and
returns them as an unordered pair.  The separator is the runtime marker for the
remote handoff tracked statically by the $\SDrop$ and $\SAcq$ constructors in
\cref{fig:process-typing}.

Rules \RuleName{R-Drop} and \RuleName{R-Acquire} move across such a separator.
Dropping removes the endpoint immediately to the left of the separator and
exposes the separator at the front of the remaining group.  Acquiring can fire
once the separator is exposed; it removes the separator and returns the
endpoint on the right.  Thus remote splitting transfers control from one
process to another only after the prefix of the original endpoint has been
consumed.

Finally, \RuleName{R-Par} and \RuleName{R-Bind} close reduction under parallel
composition and restriction.  Rule \RuleName{R-Struct} closes the relation under
process congruence, so that reductions
may use associativity, commutativity, and scope movement to expose a redex.
